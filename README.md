# Diplom_3

Метод test_follow_to_constructor_page проверяет переход по клику на «Конструктор»

Метод test_follow_to_orders_feed_page проверяет переход по клику на «Лента заказов»

Метод test_check_fluorescent_bun_form проверяет, что если кликнуть на ингредиент, появится всплывающее окно с деталями

Метод test_close_fluorescent_bun_form проверяет, что всплывающее окно закрывается кликом по крестику

Метод test_counter_ingredient проверяет, что при добавлении ингредиента в заказ, увеличивается каунтер данного ингредиента

Метод test_create_order проверяет, что залогиненный пользователь может оформить заказ

Метод test_follow_to_personal_page проверяет переход по клику на «Личный кабинет»

Метод test_follow_to_feed_orders проверяет переход в раздел «История заказов»

Метод test_exit_personal_page проверяет выход из аккаунта

Метод test_follow_to_the_password_recovery_page проверяет переход на страницу восстановления пароля по кнопке «Восстановить пароль»

Метод test_input_password_and_click_recovery_button проверяет ввод почты и клик по кнопке «Восстановить»

Метод test_checking_the_show_of_the_password_field проверяет клик по кнопке показать/скрыть пароль делает поле активным — подсвечивает его

Метод test_check_order_info_window проверяет, что если кликнуть на заказ, откроется всплывающее окно с деталями

Метод test_check_user_orders_in_orders_history, что заказы пользователя из раздела «История заказов» отображаются на странице «Лента заказов»

Метод test_update_counter_orders, что при создании нового заказа счетчик Выполнено за всё время / Выполнено за сегодня увеличивается

Метод test_check_user_order_in_job проверяет, что после оформления заказа его номер появляется в разделе В работе

**Установка зависимостей**

> `$ pip install -r requirements.txt`

**Запуск автотестов и создание HTML-отчета о покрытии**

> `$ pytest tests --alluredir=allure_results`
